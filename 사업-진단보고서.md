## 1. 🛠 프로그램 (개발/엔지니어링)

### 🔴 즉시 보완 (Critical)

#### 1.1 에러 모니터링 시스템 부재
**문제**: 매장에서 에러가 나도 미나님이 모름. 카톡으로 신고 받아야 알 수 있음.
- **해결**: **Sentry** 도입 (무료 티어 충분)
- **시간**: 1시간 셋업
- **비용**: 0원 (월 5천 이벤트까지 무료)
- **방법**: `npm i @sentry/nextjs` → 설정 가이드 따라 → 자동 추적 시작

#### 1.2 CI/CD 미설정
**문제**: 코드 push 시 빌드 실패해도 모름. 깨진 코드 라이브 반영 가능.
- **해결**: **GitHub Actions**로 빌드 체크 (Vercel은 빌드 실패 시 배포 막음 — 절반은 됨)
- **추가**: PR 머지 전 자동 빌드 + lint 체크
- **시간**: 1~2시간
- **비용**: 0원 (GitHub Actions 무료 분량 충분)

#### 1.3 분석/사용 추적 없음
**문제**: 어느 기능을 매장이 많이 쓰는지 모름. 우선순위 결정 불가.
- **해결**: **PostHog** (오픈소스, 무료) 또는 **Vercel Analytics** (간단)
- **시간**: 2~3시간 (이벤트 정의 + 코드 삽입)
- **비용**: 무료 티어로 충분 (월 100만 이벤트)

### 🟡 단기 보완 (1개월 내)

#### 1.4 테스트 부재
**문제**: 새 기능 추가 시 기존 기능 깨질 위험. 회귀 버그 방지 X.
- **해결**: **Vitest** (단위) + **Playwright** (E2E)
- **우선 테스트할 흐름**:
  - 손님 예약 신청 → 완료 페이지
  - 원장 로그인 → 예약 확정 → 매출 반영
  - 고객 모드 진입 → PIN 종료
- **시간**: 1주일 (핵심 5개 흐름)
- **비용**: 0원

#### 1.5 보안 미흡
**문제**: 
- PIN 평문 저장 (4자리, 위험 낮지만 표준 미준수)
- API rate limiting 없음 (DDoS/봇 취약)
- 약관/개인정보처리방침 X
- **해결**:
  - PIN bcrypt 해시화 (1~2일)
  - Supabase rate limiting (이미 일부 있음, Edge Function로 강화)
  - **변호사 30~50만원으로 약관 검토** (필수)

#### 1.6 백업 / 재해 복구 전략
**문제**: Supabase 장애 / 실수로 DB 날림 시 복구 매뉴얼 X.
- **해결**:
  - Supabase Pro 플랜 ($25/월) → 매일 자동 백업 + Point-in-time recovery
  - 매장별 데이터 export 기능 (사장님이 본인 데이터 다운로드)
- **시간**: 백업 자동, export 기능은 1주
- **비용**: 매장 50개 넘으면 Pro 권장

### 🟢 장기 보완 (3~6개월)

#### 1.7 성능 모니터링
- Vercel Analytics 또는 Lighthouse CI
- 페이지 로딩 시간, 코어 웹 바이탈 추적

#### 1.8 컴포넌트 시스템
- shadcn/ui 같은 컴포넌트 라이브러리 점진 도입
- 일관성 ↑, 유지보수 ↓

#### 1.9 PWA 완성도
- 192px / 512px 아이콘 추가
- 오프라인 페이지 (예약 조회만이라도)
- 푸시 알림 (예약 들어왔을 때)

#### 1.10 i18n / 다국어
- 동남아 진출 검토 시 필요
- next-intl 같은 도구 사용

#### 1.11 사진 / 동의서 데이터 효율화
- 펜 그림 / 서명 base64로 DB에 저장 중 → Storage 이전
- 사진 자동 압축 (sharp 사용)
- Storage 사용량 매장별 제한 (Pro 플랜에서)

## 2. 🎯 기획 (Product / UX)

### 🔴 즉시 보완

#### 2.1 사용자 리서치 부족
**문제**: 미나님 + 브로잉제이 외 매장 사장님 직접 인터뷰 0회. 가설 기반 개발 위험.
- **해결**: 
  - **베타 1개월 동안 매주 매장 1곳과 30분 영상통화**
  - 질문 5개 정도 표준화
  - 답변 노션/문서에 기록
- **목표**: 베타 끝날 때 5명 이상 인터뷰
- **시간**: 매주 30분 + 정리 30분
- **비용**: 0원

#### 2.2 온보딩 플로우 약함
**문제**: 회원가입 → 매장 만들기 → 그 다음 뭐? 가이드 없음. 첫 가치 체험까지 시간 길음.
- **해결**: 
  - 회원가입 직후 5단계 체크리스트:
    1. ✅ 매장 정보 완성
    2. ⏳ 시술 메뉴 1개 등록
    3. ⏳ 영업시간 설정
    4. ⏳ 첫 손님 카톡 보내기 (예약 링크)
    5. ⏳ 첫 예약 받기
  - 각 단계 누르면 해당 페이지로
  - 진행 상황 시각화
- **시간**: 1~2일
- **효과**: 베타 → 유료 전환율 ↑

#### 2.3 빈 상태 (Empty State) 가이드 부재
**문제**: 새 매장 첫 진입 시 빈 화면만 봄. 뭘 해야 할지 모름.
- **해결**: 각 페이지에 첫 액션 가이드
  - 예약 관리 (예약 0건): "손님 예약 링크는 매장 설정에서 확인하세요"
  - 고객 관리 (고객 0명): "예약이 들어오면 자동으로 등록돼요. 직접 추가도 가능"
  - 매출 관리 (매출 0): "시술 완료 처리하면 매출이 잡혀요"
- **시간**: 2~3시간 (각 페이지 빈 상태 다듬기)
- **비용**: 0원

### 🟡 단기 보완

#### 2.4 로드맵 / 기능 요청 시스템 부재
**문제**: 사장님 의견 → 카톡 → 미나님 머리 → 잊어버림.
- **해결**:
  - **Canny.io** 또는 **Notion 공개 페이지**
  - 사장님이 직접 요청 / 다른 사장님이 voting
  - 미나님이 우선순위 결정 + 진행 상황 공개
- **시간**: 1일
- **비용**: Canny 무료 또는 월 $50 / Notion 무료
- **효과**: 사장님 만족도 ↑ + 우선순위 명확

#### 2.5 NPS / 만족도 측정 미진
**문제**: 매장이 진짜 만족하는지 정량 데이터 X.
- **해결**: **분기별 NPS 설문** (10점 만점)
  - 0~6: Detractor (이탈 위험)
  - 7~8: Passive
  - 9~10: Promoter (추천 가능)
- 1문제 + 자유 응답
- 카톡 또는 이메일로 발송
- **시간**: 30분 / 분기
- **비용**: 0원

#### 2.6 사용자 세분화 X
**문제**: 모든 매장을 동일하게 다룸. 활성/비활성/이탈 위험 구분 X.
- **해결**: 매장 분류 메트릭 정의
  - **Power User**: 주 5일+ 사용, 예약 많음
  - **Regular**: 주 2~3일 사용
  - **Light**: 주 1일 미만
  - **At Risk**: 14일+ 미사용 → 카톡으로 도움 제안
  - **Churned**: 30일+ 미사용 + 결제 끊김
- /admin 대시보드에 표시
- **시간**: 2~3일
- **비용**: 0원

#### 2.7 디자인 시스템 / UX 가이드라인 X
**문제**: 페이지마다 약간씩 다른 패턴. 일관성 ↓.
- **해결**: 1페이지 디자인 가이드라인
  - 색상 / 폰트 / 버튼 스타일 / 모달 패턴 / 에러 메시지 톤
  - 새 페이지 만들 때 참고
- **시간**: 1일
- **비용**: 0원

### 🟢 장기 보완

#### 2.8 A/B 테스트 인프라
- PostHog 무료 기능에 포함됨
- 가격 페이지, 회원가입 폼 등 실험

#### 2.9 고객 성공 (Customer Success) 시스템
- 신규 매장 30일 내 첫 예약 못 받으면 자동 카톡 알림
- "도움 필요하세요?" 자동 메시지

#### 2.10 사용 분석 대시보드
- /admin에 매장별 사용 패턴 분석
- 어느 페이지에 오래 머무는지, 어디서 이탈하는지

## 3. 📣 마케팅 + 영업

### 🔴 즉시 보완

#### 3.1 브랜드 자산 부재
**문제**: 로고 X, 브랜드 가이드 X, 톤앤매너 X. "BrowChart"만 있음.
- **해결**:
  - **로고 제작**: Canva (무료) 또는 크몽/숨고 (10~20만원)
  - **브랜드 가이드 1페이지**: 색상 / 폰트 / 톤 / 사용 / 비사용 예시
  - **워드마크 3종**: 풀, 심볼, 모노크롬
- **시간**: 외주 1주, 자체 1일
- **비용**: 0~30만원

#### 3.2 인스타그램 계정 부재
**문제**: 마케팅 채널 0. 매장이 어디서 BrowChart를 알게 되나?
- **해결**: 
  - 인스타 비즈니스 계정 개설 (@browchart 또는 @browchart_kr)
  - 주 2~3회 포스팅:
    - 사용 팁
    - 사장님 후기
    - 신기능 안내
    - 비포/애프터 (매장 운영 디지털화 사례)
- **시간**: 주 2~3시간
- **비용**: 0원 (광고 빼면)

#### 3.3 랜딩 페이지 (`/`) 약함
**문제**: 단순한 페이지. 사장님이 회원가입하고 싶게 만드는 임팩트 ↓.
- **해결**: 랜딩 페이지 개편
  - **히어로**: "반영구 매장 관리, BrowChart 하나로" + 영상 / 이미지
  - **3대 가치**: 시간 절약 / 손님 만족 / 매출 증가
  - **사회적 증거**: 사용 매장 수, 후기, 강사 추천
  - **기능 미리보기**: 캘린더 / 차트 / 사진 비교 GIF
  - **명확한 CTA**: "14일 무료 시작" 버튼 큼
  - **FAQ**: "콜라보살롱과 차이?", "기존 데이터 옮길 수 있나?"
- **시간**: 1~2주
- **비용**: 0원 (디자이너 외주 시 30~100만원)

### 🟡 단기 보완

#### 3.4 후기 콘텐츠 (Social Proof) 부재
**문제**: 신규 매장이 "이거 진짜 쓸 만한가?" 의구심 가짐.
- **해결**: 
  - 베타 매장 1~2곳 후기 영상 (5분짜리)
  - 인스타 캐러셀 / 릴스
  - 랜딩 페이지에 게재
- **시간**: 영상 촬영 2~3시간 / 편집 1~2일
- **비용**: 0원 ~ 외주 50만원

#### 3.5 이메일 마케팅 시스템 X
**문제**: 매장 / 잠재 고객과 정기 소통 채널 X.
- **해결**: **Stibee** (한국 서비스, 무료 티어 충분) 또는 **Mailchimp**
  - 가입 시 환영 메일 시리즈 (5일치 자동)
  - 월 1회 뉴스레터 (신기능 + 사용 팁)
  - 14일 무료 종료 전 알림
- **시간**: 1주 (텍스트 작성 + 자동화 셋업)
- **비용**: 1,000명까지 무료

#### 3.6 SEO 미고려
**문제**: 검색 엔진에서 "반영구 매장 SaaS" 검색 시 BrowChart 안 나옴.
- **해결**:
  - 메타 태그 (title, description) 페이지별
  - sitemap.xml 자동 생성
  - 블로그 콘텐츠 (10개 키워드 글) — "반영구 매장 운영 팁" 등
- **시간**: 메타 1일, 블로그 콘텐츠 1편 / 주
- **비용**: 0원

#### 3.7 추천 / 레퍼럴 시스템 X
**문제**: 가장 강력한 마케팅 = 사용자 입소문. 인센티브 X.
- **해결**:
  - 추천 코드 시스템 (이미 브로잉제이용 만들 거잖아요)
  - 추천한 사장님 → 1개월 무료
  - 가입한 사장님 → 첫 달 50% 할인
- **시간**: 1주
- **비용**: 추천당 인센티브 ~9,500원 (=구독료의 절반 1개월 분 + 무료 1개월)

#### 3.8 분기별 영업 활동 캘린더 X
**문제**: 마케팅 / 영업 활동이 임시방편.
- **해결**: 분기 영업 캘린더 한 장
  - 1월: 신년 프로모션
  - 3월: 봄맞이 단골 관리 콘텐츠
  - 5월: 어버이날 선물 매장 마케팅
  - 7월: 박람회 참여 (서울국제뷰티엑스포)
  - 9월: 추석 휴무 안내 자동화
  - 12월: 연말 정산 매출 분석
- **시간**: 1일
- **비용**: 0원

### 🟢 장기 보완

#### 3.9 박람회 / 오프라인 마케팅
- 서울국제뷰티엑스포 (매년 가을)
- 부스 비용 100~300만원
- 매장 사장님들 직접 만남

#### 3.10 협회 / 업계 단체 제휴
- 한국반영구메이크업협회 (실제 있는지 확인 필요)
- 대량 가입 패키지 협상

#### 3.11 콘텐츠 마케팅 풀 라인업
- 블로그 (SEO)
- 유튜브 (튜토리얼)
- 뉴스레터 (구독자)
- 인스타 (브랜드)

## 4. 👤 대표자 (CEO / Leadership)

### 🔴 즉시 보완

#### 4.1 사업자 등록 미완료
**문제**: 정식 운영 / 매출 발생 / 결제 받기 위한 기본 인프라 X.
- **해결**: **개인사업자 등록**
  - 홈택스 (www.hometax.go.kr) 30분
  - 업종: "정보통신업" 또는 "소프트웨어 자문 및 개발업"
  - 무료 (수수료 X)
- **시간**: 30분
- **비용**: 0원

#### 4.2 약관 / 개인정보처리방침 부재
**문제**: 개인정보보호법 위반 위험. 손님 데이터 처리 명시 X.
- **해결**: **변호사 검토** (필수)
  - 한국법령원 / 로톡 / 헬프미 등에서 SaaS 약관 패키지
  - 비용 30~80만원 (한 번 만들면 평생 사용)
- **시간**: 변호사 1~2주
- **비용**: 30~80만원

#### 4.3 세무 / 회계 시스템 X
**문제**: 매출 발생 시 세금 신고, 영수증 발행 등 복잡.
- **해결**: **세무사 위탁**
  - 영세 사업자 대상 월 5~10만원
  - 부가세 / 종합소득세 / 원천세 다 처리
  - **자비스(jobis.co)**, **삼쩜삼** 같은 자동화 서비스 (월 1~3만원)
- **시간**: 30분 (계약)
- **비용**: 월 1~10만원

### 🟡 단기 보완

#### 4.4 자기 시간 관리 / 번아웃 방지
**문제**: 1인 사업자 = 모든 일에 끌려다니다 번아웃 위험.
- **해결**: **요일별 역할 분담**
  - 월: 개발 (오전) / 고객 응대 (오후)
  - 화: 영업 / 미팅
  - 수: 마케팅 / 콘텐츠
  - 목: 개발 (집중)
  - 금: 회고 + 다음 주 계획
  - 주말: 휴식 (꼭!)
- **시간 관리 도구**: Notion, Todoist, Cal.com
- **비용**: 0원

#### 4.5 멘토 / 네트워크 부재
**문제**: SaaS 운영은 처음. 시행착오 시간 / 비용 큼.
- **해결**: 
  - **한국 SaaS 커뮤니티** 참여 (스타트업위클리, 디스콰이엇, 매스프레소)
  - **EO 같은 미디어** 구독 (사례 학습)
  - **링크드인**: 다른 SaaS 대표들 팔로우 + 메시지
  - **멘토 1명** 찾기 (반영구 X, 일반 SaaS 대표) — 분기 1회 30분
- **시간**: 주 2~3시간 (학습 + 네트워크)
- **비용**: 0원 ~ 멘토 식사비

#### 4.6 1년 / 3년 / 10년 비전
**문제**: 매일 작업에 몰두 → 큰 그림 잃기 쉬움.
- **해결**: **비전 한 페이지 적기** + 매월 검토
  ```
  1년 후: 매장 100개, 월 매출 200만원
  3년 후: 매장 500개, 월 매출 1,000만원, 직원 2~3명
  10년 후: 한국 반영구 SaaS 1위, 동남아 진출
  ```
- 매월 1일 = 비전 검토일
- **시간**: 1시간
- **비용**: 0원

#### 4.7 회사 가치 / 문화 정의 X
**문제**: 직원 채용 시 어떤 사람을 뽑을지 기준 X.
- **해결**: 핵심 가치 3개 정의
  - 예: "1. 사용자 우선 / 2. 빠른 반복 / 3. 솔직한 소통"
  - 채용 / 의사결정 / 운영 시 가이드
- **시간**: 반나절
- **비용**: 0원
